## approach

**Less, but better** — every line of code is a liability. The best solution is usually the simplest one that actually works.

**Ship and iterate** — perfect is the enemy of shipped. Get it out, watch it run, make it better.

**Open by default** — code should be readable, forkable, and improvable by anyone who needs it.
